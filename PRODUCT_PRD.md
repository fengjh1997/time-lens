# Time Lens (星辰透镜) - 产品需求文档 (PRD)

## 1. 产品概述
Time Lens 是一款专注于“心流能量”管理的高级时间记录与规划工具。它通过“能效评分”而非仅仅是“任务清单”来衡量时间的价值。

## 2. 核心功能
### 2.1 智能视图系统
- **一周规划 (Week View)**: 全局视角，支持任务预设与历史能效热力图。
- **每日焦点 (Day View)**: 垂直时间轴，支持精确到 15min 的心流打卡。
- **休眠折叠**: 自动折叠非工作时段 (23:00-09:00)，聚焦活跃时间。

### 2.2 能量记录与评分
- **星辰能量 (Star Energy)**: 提供 0.25 - 1.0 的四档星级评分，量化心流质量。
- **互动式番茄钟 (v6.2.0)**: 记录模态框内置 25 分钟番茄钟，仅在当前时间段显示位置。专注完成自动生成打卡标签 (🍅)。
- **额外增益 (Bonus Time)**: 捕捉计划外的闪光时刻，计入全天能量总和。

### 2.3 云端同步与多端访问 (v6.1.0+)
- **Supabase 强力驱动**: 支持跨设备实时同步记录、标签与个人偏好。
- **安全身份验证**: 支持邮箱登录与 GitHub 第三方快捷登录。

### 2.4 个性化设计系统
- **动态主题引擎**: 支持 琥珀金、翡翠绿、紫罗兰、蔚蓝 四大色彩体系。
- **沉浸式感官**: 全站玻璃拟态 (Glassmorphism) 设计，适配极致深色/浅色模式。
- **Super Round 美学**: 极致圆润的 UI 组件，提供高端、亲和的交互体验。
- **全景热力图 (v6.2.0)**: 月/年视图升级为基于主色的动态热力图，实时反映能量分布。
- **标签洞察 (v6.2.0)**: 数据洞察页支持特定标签的全域能量热力分析。

## 3. 数据架构与向前兼容规划
为了保证应用在版本大幅迭代后（如 v7.x, v8.x）数据的稳定性，制定以下技术方案：
- **Schema 软升级策略**: 数据库字段采用“增量添加”模式。如果新功能需要新字段，原有的 `blocks` 或 `settings` 表将保持旧字段兼容，防止破坏历史数据。
- **JSONB 扩展性**: 对复杂的标签（Tags）和未来可能增加的元数据，优先考虑存储在 JSONB 字段中，实现 Schema-less 动态扩展。
- **Supabase 稳定性**:
  - **RLS 策略隔离**: 严格的 Row Level Security (RLS) 确保任何人只能读取/修改自己的数据。
  - **版本快照**: 在进行重大数据库迁移（Migration）前，将强制本地备份与云端数据快照。
  - **API 兼容层**: 前端代码中封装 `useSync` 钩子，屏蔽后端表结构优化带来的直接影响。
  - **多端数据冲突处理**: 采用 "Last Write Wins" (LWW) 与 "Optimistic UI" 结合，确保弱网环境下记录不会丢失，并在恢复连接时自动增量同步。

## 4. 技术栈
- **Frontend**: Next.js 14, React, Tailwind CSS.
- **State**: Zustand (with Persist Middleware).
- **Backend**: Supabase (Auth, Database, RLS Security).
- **Icons**: Lucide React.
- **Deployment**: Vercel (CI/CD via GitHub).

## 4. 版本标准
- **语义化版本**: 严格遵循 `vX.Y.Z` 模式。
- **备份策略**: 仅在完成核心里程碑（Minor/Major 版本）后推送到 GitHub。
